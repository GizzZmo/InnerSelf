## Getting Started

- Fork this repository and clone your fork locally.
- Create a new branch for your feature or bugfix.
- Follow existing code style and add docstrings/comments where appropriate.
- Write or update tests as needed.

## Pull Requests

- Ensure your branch is up to date with main.
- Open a pull request with a descriptive title and explanation of your changes.
- Reference related issues in your PR description.
- Participate in the code review process by responding to comments and making requested changes.

## Issue Reporting

- Search existing issues before opening a new one.
- Include a clear and descriptive title and as much relevant information as possible.
